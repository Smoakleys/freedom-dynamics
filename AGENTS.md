# Freedom Dynamics agent rules

These are durable project instructions from Bridger. Read `HANDOFF.md`,
`PLAN.md`, `CONVERSATION_LOG.md`, and `docs/VISUAL_REVIEW.md` before making
project changes.

## Continuity

- Update `HANDOFF.md` and append to `CONVERSATION_LOG.md` at the start of a new
  directive, after every material implementation/review milestone, and after
  deploy. Another agent must be able to resume without the chat transcript.
- Never describe work as started unless the corresponding file/tool work begins
  in the same turn.

## Visual standard

- Hold the game to a high premium-product bar. "Technically improved" is not a
  pass when the full composition still feels clunky, noisy, or thrown together.
- Prefer clear hierarchy, intentional spacing, smooth silhouettes, and a small
  number of strong signals over simultaneous labels, borders, patterns, units,
  effects, and HUD text.
- Units are fixed world objects and must be substantially smaller than the
  July-19 Bold Board deployment; they disappear at strategic altitude.
- Borders must read as genuinely smooth designed curves at every supported zoom,
  not softened traces of a cell grid.
- The primary money value belongs centered at the top of the battlefield HUD.
- Mobile map control is first-class: predictable one-finger pan, focal pinch
  zoom, no page-gesture interference, and clear feedback/help for controls.
- Run the visual gate at far, mid, and close zoom on mobile dimensions. Review
  the whole UI composition, not only the map texture.

## Collaboration

- Bridger welcomes preference questions. Ask only specific, decision-shaped
  questions (for example, two or three concrete alternatives with the visual or
  interaction consequence), never vague style prompts.
- Direct negative feedback supersedes earlier approval. Preserve useful lessons,
  but do not defend a direction that no longer meets the bar.
